# 项目更新日志 (CHANGELOG)

## 项目简介 (Project Introduction)
**Automated-GMCQs-Answering** (高中地理多项选择题自动化问答系统) 
本项目旨在通过自然语言处理(NLP)和三支形式概念分析(Three-way Concept Lattices)技术，实现地理单选与多项选择题的自动化解析和作答。系统通过提取题干中的地理特征、构建基于领域词典的实体关联矩阵、生成三支概念格，并最终提取出高置信度的地理推断规则，以此来自动映射并推断出正确选项。

## 2026-03-22 更新记录

### 1. 多选题解题逻辑增强
* **重写选项评分与答案决策流程**: 在 `batch_evaluate.py` 中引入“直接证据分 + 规则证据分 + 组合证据分”的综合打分框架，并据此分别实现单选题与多选题的答案推断逻辑。
* **增强题干与选项的语义桥接**: 新增题干片段拆分、关键词净化、选项片段支持度、领域词典上下级桥接、规则结论到选项标签映射等机制，提高多选题对同义表达和拆分表述的识别能力。
* **加入错误选项抑制机制**: 对包含“一定”“完全”“不会”“直接决定”等绝对化表述的选项增加风险惩罚，减少多选题中看似合理但实际错误选项被高分选中的情况。
* **兼顾单选题准确率**: 新增否定式题干识别（如“不正确”“不属于”“错误的是”等），在单选题中支持“低分优先”判定，避免为了提升多选题而破坏原有单选题判断逻辑。
* **修复多选题决策域过滤问题**: 调整 `entity_attribute.py` 中决策属性组合的生成方式，不再错误地只保留分类名以“型”结尾的选项，确保真实选项全部参与组合推断。

### 2. 多选领域词典增强与覆盖率补齐
* **新增词典增强脚本 `enrich_multiple_dictionaries.py`**: 自动融合 `dict_multiple`、`dict_single`、多选题 JSON 选项文本及解析片段，批量回填和扩展多选词典。
* **批量扩充 6 个多选领域词典**: 更新 `dict_multiple/Climatology.csv`、`GPP.csv`、`HEG.csv`、`Hydrology.csv`、`SV.csv`、`TG.csv`，显著补齐多选场景下的选项短语、解释片段和上下位概念。
* **补全原先覆盖薄弱的词典**: 此前内容严重不足的 `dict_multiple/SV.csv` 与 `dict_multiple/TG.csv` 已补全，多选题可用概念显著增加。
* **修复重复关键字覆盖问题**: `geo_keyword_extractor.py` 现支持合并同一关键字的多行上级/下级信息，而不是简单以后者覆盖前者。
* **完成一次多选选项覆盖检查**: 6 个学科类别的多选题选项文本已实现 100% 词典覆盖，减少“选项在词典中不存在”导致的匹配失真。

### 3. 评测层缓存与规则提取性能优化
* **`entity_attribute.py` 缓存优化**: 为分词结果、字符集合和相似度计算增加缓存，并对候选概念先进行字符重合预筛，减少无效相似度比较。
* **`geo_keyword_extractor.py` 加载优化**: 为领域词典和同义词表增加缓存，并将逐行遍历从 `iterrows()` 改为 `itertuples()`，降低词典初始化开销。
* **`extract_three_way_rules_enhanced.py` 向量化**: 将规则置信度计算中的大量 `pandas.iloc` 逐格访问改为 `numpy` 向量化处理，减少规则提取阶段的嵌套循环开销。
* **`batch_evaluate.py` 初始化优化**: `jieba.add_word()` 改为单进程仅初始化一次，避免批量评测时重复加载自定义词。
* **代表性测速结果**: 在早期缓存与向量化优化阶段，气候类多选单题分析约由 `2.964s` 降至 `1.680s`，气候类单选单题分析约由 `1.093s` 降至 `0.181s`。

### 4. 三支概念格 (`threeWcl`) 计算链路提速
* **重构 `BasicCL` 的复用方式**: `threeWcl.py` 与 `util/CL.py` 改为复用同一个 `BasicCL` 实例，避免在内层循环中反复创建临时对象。
* **为核心下运算增加缓存**: 在 `util/basic.py` 中为 `intersectForObject()` 增加基于 `(bpc, index_tuple)` 的缓存，并预计算 `bpcObj/bpcAttr` 的 tuple/set 视图，减少重复交集运算与排序。
* **优化 AE / OE 双层循环**: `threeWcl.py` 中新增概念视图预处理和 tuple->set 复用逻辑，显著降低 `bp1 x bp2` 双重遍历中的重复集合构造成本。
* **修复 `Pair` 哈希实现**: `util/vo.py` 中将 `Pair.__hash__` 从不合理的逻辑与写法改为 `hash((self.l, self.r))`，提升大量集合判重场景下的性能稳定性。
* **代表性测速结果**: 中等规模形式背景下，`process_formal_context()` 耗时约由 `0.434s` 降至 `0.052s`；在真实题目生成的条件背景上，旧版同场景运行 `35s+` 仍未结束，新版约 `8.5s` 完成。

### 5. 三支概念合并 (`merge_concepts.py`) 重写提速
* **消除 AE x OE 的全量两两扫描**: 原实现对每个 `AE` 顺序扫描全部 `OE`，复杂度为 `O(|AE| * |OE|)`，在大规模概念下会出现数量级爆炸。
* **改为“流式解析 + 哈希索引”方案**: 新实现将概念解析为可哈希的有序 `tuple`，并建立 `(X, A)` 与 `(X, B)` 两类索引，使匹配过程降为直接查表。
* **保留原始顺序语义**: 索引中记录 OE 概念的原始位置，尽量保持与旧版“从前往后扫描 OE，找到第一个匹配项即停止”一致的匹配行为。
* **代表性测速结果**: 在 `AE=32504`、`OE=33096` 的真实结果文件上，新的 `merge_concepts.main()` 总耗时约 `1.98s`；旧实现运行 `35s+` 仍未完成。

### 6. 验证与回归
* **端到端验证通过**: 使用 `multiple_questions/Climatology.json` 第 1 题进行了完整流水线验证，系统在当前优化后仍正确预测 `['A', 'B', 'C']`。
* **语法检查通过**: 已通过 `python -m py_compile threeWcl.py util/basic.py util/CL.py util/vo.py merge_concepts.py` 的语法校验。
* **当前工作重点说明**: 本次更新优先解决了多选题解题逻辑、领域词典覆盖和评测耗时瓶颈；全量题库的完整回归准确率评测仍需在后续补跑。

## 2026-03-08 更新记录

### 1. 数据结构与题库规范化重构
* **题库 JSON 化**: 将原有的纯文本题库全面迁移为结构化的 JSON 格式，并按照地理学科类别拆分为多个题库文件（如 `single_Climatology.json`, `single_Geographical Processes and Principles.json` 等）。
* **词典重命名与规范化**: 编写脚本将原有的“词典*.csv”批量重命名为英文“dic*.csv”，解决跨平台编码及规范命名问题。

### 2. 核心流水线重构与性能优化
* **全新主控调度器 `batch_evaluate.py`**:
  * 移除了冗余且计算开销极大的旧版 `auto_run.py` 与 `main.py`（原版使用 `subprocess` 循环调用子进程）。
  * 实现了将所有题目放入同一个 Python 进程中循环处理的架构，使 `jieba` 分词和大型领域词典等耗时资产**仅需加载一次**。
  * 增加了针对前道题目生成的历史文本缓存清理机制，防止测试集之间的结果互相污染。
* **修复跨平台路径 Bug**: 移除了 `threeWcl.py` 中写死的 Windows 绝对路径（`C:\Users\...`），改为使用 `os.path.abspath(__file__)` 动态获取当前相对路径，使项目顺利跑通于 macOS 及 Linux。

### 3. 新增计算功能：自动化推断与准确率计算 (Inference Module)
* 在 `batch_evaluate.py` 尾部新增了选项推断模块。
* 支持根据 `extract_three_way_rules_enhanced.py` 吐出的“最高置信度和强度规则”，自动匹配 A/B/C/D 四个选项。
* 实现了**双模相似度匹配机制**（基于 jieba 分词的语义交集计算 + 基于字符级的重合度计算），从而容错选项中字面不完全一致的推断（如“夏季干热”自动关联到“干燥”选项）。
* 在所有题目分析结束时，自动输出 `batch_evaluation_summary.txt` 整体准确率报告。

### 4. 算法与逻辑 Bug 修复
* 修复了 `entity_attribute.py` 中 `get_top_similar_concepts()` 的一个致命逻辑 Bug：此前当题干特征与词典所有词打分均为 `0.0` 相似度时，代码依然会强行把排序榜首的不相关词汇牵扯进来纳入实体关联类中（导致如“喀斯特地貌”误入毫不相干的题目）。现在增加了 `> 0.0` 的绝对限制。

### 5. 垃圾文件清理
* 移除了项目重构后不再需要的废弃文件，包括：`auto_run.py`, `main.py`, `rename_dicts.py`, `test_questions.csv`。
